# Analisis de notebooks de programacion

## Alcance
Usar cuando el notebook trate sobre Python base, programacion, algoritmos, estructuras de datos, funciones, POO, manejo de archivos, APIs, scripts, automatizacion o fundamentos de software.

## Revision especifica
Ademas del analisis generico, revisar:

- imports y dependencias;
- variables no definidas;
- orden de ejecucion;
- funciones y responsabilidades;
- estructuras de datos usadas;
- complejidad si aplica;
- casos de prueba;
- entradas y salidas;
- manejo de errores;
- legibilidad y comentarios;
- estilo consistente.

## Reglas
- Corregir con cambios minimos.
- No reescribir todo el notebook si basta corregir una celda.
- Proponer ejemplos de prueba simples.
- Preferir claridad sobre soluciones innecesariamente sofisticadas.
- Explicar por que el codigo funciona y que casos cubre.

## Salida esperada
Codigo listo para copiar, comentarios pedagogicos, pruebas minimas y advertencias sobre errores frecuentes.
