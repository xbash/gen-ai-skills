# Pruebas, calidad y seguridad

- Elegir pruebas segun riesgo: unitarias, integracion, contrato, e2e, regresion, smoke y carga.
- Cubrir casos felices, bordes, errores, permisos y regresiones.
- Mantener tests deterministas y legibles.
- En seguridad, validar autenticacion, autorizacion, secretos, dependencias, input, output encoding, SSRF, XSS, SQLi, CSRF, path traversal y logs.
- No esconder errores; entregar mensajes utiles sin filtrar datos sensibles.
- Revisar supply chain, versiones, licencias y vulnerabilidades.
