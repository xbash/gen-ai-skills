# Backend, frontend y APIs

- En backend, definir contrato, autenticacion, autorizacion, validacion, errores, idempotencia, logs y observabilidad.
- En APIs, cuidar versionado, compatibilidad, paginacion, filtros, limites, errores y seguridad.
- En workers/colas, considerar retries, deduplicacion, orden, poison messages e idempotencia.
- En frontend, seguir patrones existentes, accesibilidad, estado, formularios, loading, error, vacio y responsive.
- Validar entradas en cliente y servidor.
- No exponer secretos ni datos sensibles en logs, UI o respuestas.
