# Checklist - Ingenieria de software

- [ ] Objetivo y alcance claros.
- [ ] Cambio pequeno y localizado.
- [ ] Contratos, entradas y salidas validados.
- [ ] Errores, logs y permisos revisados.
- [ ] Seguridad y secretos protegidos.
- [ ] Datos, migraciones y rollback considerados.
- [ ] Tests o prueba de humo definidos.
- [ ] Compatibilidad y regresiones revisadas.
- [ ] Documentacion o notas operativas actualizadas si aplica.
- [ ] Riesgos y mejoras opcionales separados.
