# Ingeniería de software

Dominio para diseñar, construir, corregir, revisar, probar, documentar, desplegar y mantener software con rigor académico-técnico y orientación práctica.

## Archivos

| Archivo | Uso recomendado |
| --- | --- |
| `00_instrucciones_base_dev.md` | Instrucción personalizada base para ingeniería de software y desarrollo aplicado. |
| `01_diseno_arquitectura_reglas.md` | Diseño, arquitectura, patrones, modularidad, algoritmos, estructuras de datos y trade-offs. |
| `02_backend_api_reglas.md` | Backend, APIs, workers, colas, autenticación, autorización e integraciones. |
| `03_frontend_web_reglas.md` | Frontend web, componentes, formularios, estado, accesibilidad, responsive y rendimiento. |
| `04_bases_datos_sql_reglas.md` | Bases de datos, SQL, persistencia, migraciones, transacciones, índices y ORM. |
| `05_pruebas_calidad_reglas.md` | Pruebas, calidad, regresión, refactorización y mantenibilidad. |
| `06_devops_ci_cd_reglas.md` | CI/CD, contenedores, releases, artefactos, ambientes, despliegue y operación. |
| `07_seguridad_codigo_reglas.md` | Seguridad de aplicaciones, código defensivo, secretos, dependencias, errores y logs. |
| `08_checklist_codigo_dev.md` | Checklist transversal para revisar o entregar código, scripts, servicios y sistemas. |
| `90_estructura_y_usos.md` | Guía de estructura y uso del dominio en distintos LLMs. |

## Recomendación práctica

Para un proyecto en ChatGPT, Claude, Gemini, Qwen o GLM, usa `00_instrucciones_base_dev.md` como archivo principal y agrega los archivos específicos según la tarea.

Para ChatGPT se recomienda usar `pack-chatgpt`, que fusiona el dominio en 7 archivos y deja margen para código, logs, requisitos o capturas.
