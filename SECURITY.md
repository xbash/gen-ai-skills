# Security

Este repositorio contiene instrucciones para asistentes de IA. Algunas areas pueden impactar seguridad, salud, derecho, privacidad, finanzas, identidad visual o sistemas productivos.

## Principios de seguridad

- No usar las instrucciones para dano, intrusion, fraude, evasion legal, manipulacion, doxxing, acoso o falsificacion.
- No exponer secretos, credenciales, datos personales, informacion financiera sensible ni rutas privadas.
- No presentar orientacion general como diagnostico medico, asesoria legal, asesoria financiera personalizada o instruccion profesional definitiva.
- En seguridad operacional y AppSec, mantener foco defensivo, autorizado y documentado.
- En fotografia, no alterar identidad, evidencia, documentos o contexto visual de forma enganosa.

## Reporte de problemas

Si detectas una instruccion que pueda facilitar dano, abuso, desinformacion, diagnostico indebido, evasion legal, exposicion de secretos o manipulacion enganosa, corrige el archivo afectado y deja trazabilidad en el changelog o en la descripcion del cambio.

## Criterios minimos para cambios seguros

- Incluir limites de alcance.
- Indicar cuando derivar a profesionales o fuentes oficiales.
- Evitar instrucciones operativas peligrosas si no hay autorizacion explicita.
- Preferir mitigacion, defensa, evaluacion, auditoria y educacion.
- Declarar incertidumbre cuando falte contexto o evidencia.
